# SafeShare - 잠재적 문제점 및 대응 방안

## 1. 보안 문제

### 1-1. 악성 PDF 업로드
**문제**: PDF에 악성 JavaScript나 exploit이 포함될 수 있다.
**대응**:
- pdf.js는 기본적으로 JavaScript 실행을 차단함 (별도 설정 불필요)
- R2에 저장 시 Content-Type을 application/pdf로 고정
- 매직 바이트 검증으로 위장 파일 차단
- 향후: ClamAV 연동 (VPS 전환 시)

### 1-2. 불법/부적절 콘텐츠 업로드
**문제**: 서비스가 불법 자료 유통에 악용될 수 있다.
**대응**:
- 업로더 IP 기록 (법적 요청 시 제공 가능)
- 파일 크기 제한 (50MB)
- Rate limiting으로 대량 업로드 차단
- 신고 기능 추가 고려 (Phase 2)
- 이용약관에 면책 조항 명시

### 1-3. 공유 링크 무차별 대입 (Brute Force)
**문제**: shareId를 무작위로 넣어서 다른 사람 문서에 접근 시도.
**대응**:
- nanoid(8)은 약 2조개(2^41) 경우의 수 → 무차별 대입 비현실적
- 존재하지 않는 링크에 대해 일관된 404 응답 (타이밍 공격 방지)
- 향후: 비밀번호 보호 옵션 추가 가능

### 1-4. R2 직접 접근
**문제**: R2 버킷 URL이 노출되면 만료된 파일도 접근 가능.
**대응**:
- R2 버킷은 퍼블릭 접근 차단 (기본 설정)
- 반드시 API를 통해서만 파일 서빙 (API에서 만료 체크)
- presigned URL 사용 시에도 만료 시간을 짧게 설정 (1시간)

## 2. 성능 문제

### 2-1. 대용량 PDF 업로드 실패
**문제**: Cloudflare Workers는 요청 본문 100MB 제한, 실행 시간 제한이 있다.
**대응**:
- 50MB 파일 크기 제한 설정
- 향후 대용량 지원 필요 시 presigned URL 방식으로 전환
  (브라우저 → R2 직접 업로드, Workers는 메타데이터만 처리)

### 2-2. PDF 뷰어 성능 (모바일)
**문제**: 페이지가 많거나 이미지가 많은 PDF는 모바일에서 느릴 수 있다.
**대응**:
- pdf.js의 lazy rendering 활용 (현재 보이는 페이지만 렌더링)
- 로딩 중 스켈레톤 UI 표시
- 모바일에서는 줌 레벨 제한으로 메모리 절약

### 2-3. 동시 접속 폭주
**문제**: 선생님이 학부모 단체방에 링크를 보내면 수백 명이 동시 접속.
**대응**:
- Cloudflare CDN이 정적 자산 캐싱 처리
- R2 파일 응답에 Cache-Control 헤더 설정 (private, max-age=3600)
- pdf.js 자체는 CDN에서 로드
- 뷰어 페이지는 SSG(정적 생성) 불가이므로 Edge SSR로 빠르게 응답

## 3. UX 문제

### 3-1. 카카오톡 인앱 브라우저 호환성
**문제**: 카카오톡은 자체 인앱 브라우저를 사용하며 일부 Web API가 제한됨.
**대응**:
- 클립보드 API (navigator.clipboard) 미지원 시 fallback (execCommand)
- pdf.js는 인앱 브라우저에서도 동작 확인됨
- "외부 브라우저로 열기" 안내 문구 추가 (문제 발생 시)
- 테스트: 카카오톡 인앱, 삼성 인터넷, Chrome, Safari

### 3-2. iOS Safari PDF 핸들링
**문제**: iOS Safari는 PDF를 자체 뷰어로 열려는 경향이 있음.
**대응**:
- Content-Type: application/pdf + Content-Disposition: inline으로 서빙
- pdf.js 사용으로 브라우저 기본 PDF 뷰어 우회
- 다운로드 시에만 Content-Disposition: attachment

### 3-3. 작성자가 링크를 잃어버림
**문제**: 링크 생성 후 복사 안 하고 페이지를 닫으면 다시 찾을 수 없음.
**대응**:
- 링크 생성 직후 강조된 안내: "지금 복사해두세요"
- 브라우저 localStorage에 최근 생성 링크 목록 임시 저장 (같은 기기에서만)
- 향후: 카카오 로그인 시 내 문서함에서 관리

### 3-4. 만료 기간 인지
**문제**: 열람자가 만료 기간을 모르고 나중에 접속하면 이미 삭제됨.
**대응**:
- 뷰어 상단에 만료일 명시
- 만료 3일 전부터 경고 배너 색상 변경
- 다운로드 허용 시 "필요하면 다운로드해두세요" 안내

## 4. 법적/운영 문제

### 4-1. 개인정보 포함 문서
**문제**: 학부모 연락처, 학생 정보 등이 포함된 문서가 업로드될 수 있음.
**대응**:
- 자동 만료 삭제로 데이터 보유 최소화
- 이용약관에 "개인정보 포함 문서 업로드 시 업로더 책임" 명시
- 파일 암호화 저장은 Phase 2에서 고려

### 4-2. 서비스 안정성
**문제**: Cloudflare 무료 티어에 의존하면 서비스 중단 리스크.
**대응**:
- Cloudflare는 무료 티어도 SLA가 안정적
- 다만 R2/D1 무료 한도 초과 시 서비스 중단 가능
- 모니터링 설정: 사용량이 무료 한도 80% 도달 시 알림
- 비상 계획: Vercel + Supabase 스택으로 마이그레이션 가능

### 4-3. 저작권 문제
**문제**: 저작권 있는 자료가 업로드될 수 있음.
**대응**:
- DMCA 대응 절차 마련 (연락처 공개)
- 신고 접수 시 즉시 삭제 프로세스
- 이용약관에 저작권 침해 금지 조항

## 5. 향후 확장 시 고려사항

### Office 변환 추가 시 (Phase 2)
- 클라이언트 변환 품질 한계로 사용자 불만 가능
- 변환 전/후 미리보기 비교 UI 필요
- "변환 결과가 원본과 다를 수 있습니다" 안내 필수
- 변환 실패 시 graceful fallback (직접 PDF로 변환 안내)

### 사용자 인증 추가 시 (Phase 3)
- 카카오 로그인 OAuth 연동
- 문서 관리 대시보드 (내 문서 목록, 삭제, 만료 연장)
- 업로더별 사용량 제한

### 도메인/브랜딩
- .kr 도메인 취득 시 공무원 신뢰도 향상
- 카카오톡 OG 이미지 커스터마이징으로 공유 시 시인성 확보
